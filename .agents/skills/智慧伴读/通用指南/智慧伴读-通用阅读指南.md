# 智慧伴读 · 通用阅读指南

> 基于花叔《阅读的方法》改良版
> 适用于任何 AI 助手（Claude Code、ChatGPT、Cursor 等）

## 核心原则

智慧伴读不是"替人读书"，而是**伴人读懂**。核心方法论分三步：

---

## 第一步：狩猎法（带着问题读）

**适用场景**：你有一个具体问题，想从书中找到答案。

### 操作流程

1. **明确你的问题** — 不要问"这本书讲了什么"，要问"这本书对我当前遇到的 XX 问题有什么启发"
2. **定位猎物** — 在书中找到与你问题最相关的 3-5 个核心观点/段落
3. **苏格拉底追问** — 针对每个定位到的观点，追问：
   - 这个观点和你的问题之间是什么关系？
   - 它有没有让你想到自己经历过的某个具体场景？
   - 它和你原来的理解有什么冲突？

### 提示词模板（给 AI）

```
我正在读《书名》，我的问题是：<你的具体问题>
请用"狩猎法"帮我：
1. 从书中定位与我问题最相关的 3 个核心观点
2. 对每个观点，追问它如何链接到我的真实处境
3. 不要替我做总结，用问题引导我思考
```

---

## 第二步：费曼法（6岁解释测试）

**适用场景**：你觉得自己"懂了"，但说不清楚。

### 操作流程

1. **选一个核心概念** — 从狩猎法定位的结果中选一个
2. **用最简单的语言解释** — 假装讲给一个 6 岁孩子听，不能用术语
3. **暴露盲点** — 在解释过程中，你一定会发现"这里我其实没搞懂"
4. **迭代** — 针对盲点重新理解，再解释一次

### 提示词模板（给 AI）

```
请用"费曼法"帮我理解<概念名>：
1. 让我先试着用自己的话解释
2. 你扮演魔鬼教练，追问我的解释中模糊的地方
3. 每轮只追问一个点，不要一次性全抛出来
```

---

## 第三步：辩论法（压力测试）

**适用场景**：你想验证一个核心观点是否经得起推敲。

### 操作流程

1. **确定命题** — 把你想验证的观点写成一句话
2. **邀请对手** — 找 2-3 个跨学科"对手"来攻击这个观点
   - 实用主义者（如查理·芒格）："这在实际中行得通吗？"
   - 怀疑论者（如费曼）："你确定这不是自欺欺人？"
   - 长期主义者（如纳瓦尔）："这个观点的复利效应在哪？"
3. **作者防守** — 让原作者回应攻击
4. **裁判裁决** — 逐条评判：哪些攻击击中了要害，哪些被成功防守

### 提示词模板（给 AI）

```
请帮我辩论以下观点：
"<你的观点>"

请扮演以下角色：
1. <对手1>（如查理·芒格）— 从实用主义角度攻击
2. <对手2>（如费曼）— 从第一性原理角度攻击
3. 作者 — 防守反击
4. 裁判 — 给出打分裁决
```

---

## 三级模式速查

| 模式 | 步骤 | 耗时 | 产出 |
|------|------|------|------|
| 🏹 轻量 | 仅狩猎法 | 15-30 分钟 | 问题相关的 3-5 个观点 + 追问 |
| 🧠 标准 | 狩猎法 → 费曼法 | 30-60 分钟 | 澄清的概念 + 暴露的盲点 |
| ⚔️ 深度 | 狩猎法 → 费曼法 → 辩论法 | 60-120 分钟 | 经得起推敲的结论 + 被削弱的盲区 |

---

## 输出模板

阅读完成后，建议整理成以下结构：

```markdown
# 《书名》阅读笔记

## 我的问题
<你最初带着什么问题来读>

## 核心发现
<狩猎法定位到的 3 个关键观点>

## 澄清记录
<费曼法暴露的盲点和你的新理解>

## 压力测试结果
<辩论法的交锋记录和裁判裁决>

## 我的反思
<读完后的真实感受和下一步行动>
```

| Stage | 模块 | 说明 |

| 2 | 费曼法 | 用 6 岁解释测试暴露盲点 |
| 3 | 华山论剑 | 跨学科对手压力测试 |
